# 音楽の黄金時代とは!?

[タブレット純_音楽の黄金時代](https://www.jorf.co.jp/?program=tabjun)は、
芸人のタブレット純さんがパーソナリティを務める、
昭和歌謡を中心にした音楽トーク番組です。

![](/images/articles/76728c0ec86508/01.png)

当該番組は、毎回様々な昭和歌謡を深いウンチクと共に紹介されており、
昭和レトロファンにはたまらない悶絶構成となっております。
(今週の特集は"みんなのうた"でした!!)

そこで今回は、家に眠っている腐ったラズパイ(なんとRaspberry Pi3)を引き摺り出し、
"音楽の黄金時代"のRadiko生放送にトライします。

勿論、[Podcast](https://podcasts.apple.com/jp/podcast/%E3%83%A9%E3%82%B8%E3%82%AA%E6%97%A5%E6%9C%AC-%E3%82%BF%E3%83%96%E3%83%AC%E3%83%83%E3%83%88%E7%B4%94-%E9%9F%B3%E6%A5%BD%E3%81%AE%E9%BB%84%E9%87%91%E6%99%82%E4%BB%A3-%E3%83%9D%E3%83%83%E3%83%89%E3%82%AD%E3%83%A3%E3%82%B9%E3%83%88/id1690131460)でも放送は聞けるのですが、"そういう事"ではないのです。

# 仮想環境を用意する

腐ラズパイ(Raspberry Pi3)は、SSHで操作します。
[Raspberry Piハンズオン01_(インストール~SSH接続まで)](https://zenn.dev/link/comments/6b2483fe76cfcc)を参考にしてくださいね。

以降、"kajiru"の部分は、みなさんのユーザーIDに置き換えて読み進めてください。

```bash:command
ssh kajiru@raspberry3-desktop.local
```

古いラズパイなので、念の為Pythonのバージョンを確認しておきます。
確認できたバージョンをメモしておきます。(ここでは、3.11)

```bash:command
python3 --version
```

先ほど確認したバージョンと、同じバージョンでvenvをインストールします。

```bash:command
sudo apt install python3.11-venv
```

次に、次のコマンドで仮想環境を準備します。
今回の仮想環境は、デスクトップに用意します。
下記のコマンド実行後、デスクトップに"my_venv"というフォルダが出来ていれば成功です。

```bash:command
cd /home/kajiru/Desktop
python3 -m venv my_venv
```

最後に、仮想環境を有効にします。

```bash:command
cd /home/kajiru/Desktop/my_venv/bin
source activate
```

仮想環境を終了するときは、次のコマンドを実行します。

```bash:command
dactivate
```

ここまでの構成は次のとおりです。

```text:フォルダ構成
/home/kajiru/Desktop/
　└ my_venv <- 仮想環境をインストールしたフォルダ
```

# ライブラリのインストール

仮想環境を有効"activate"した状態で、
Radikoの再生に必要なライブラリをインストールします。

```bash:command
# Radikoの放送URLを整理して出力するライブラリ
python3 -m pip install streamlink
# Radikoをライブ放送するプレイヤー
sudo apt-get install mpv
```

ここで、mpvが利用できるオーディオデバイスを確認しておきます。
この指定を間違えると、音が流れないので注意です。
今回は、一覧の中からUSBで接続したスピーカーを利用することにします。
ここでは、"alsa/dmix:CARD=PJP10UR,DEV=0"です。

```bash:command
# mpvが利用できるオーディオデバイスの一覧
mpv --audio-device=help
```

[Radiko](https://radiko.jp/#!/top)のURLから、利用可能なライブ配信のURLを探します。
[ぎふちゃん](https://radiko.jp/#!/live/GBS)であれば、"https://radiko.jp/#!/live/GBS"という具合です。

準備ができたら、次のコマンドで実際に再生できるかを確認しましょう。
音が再生されない場合は、RadikoのURLが間違っている、オーディオデバイスの指定が間違っている等の可能性があります。

```bash:command
# Radikoを再生
/home/kajiru/Desktop/my_venv/bin/streamlink -p /usr/bin/mpv --player-args="--audio-device=alsa/dmix:CARD=PJP10UR,DEV=0" 'https://radiko.jp/#!/live/GBS' worst
```

# シェルにまとめる

これまでの一連の流れをシェルにまとめておきます。
新たに、"start.sh"というファイルを用意し、次のコードを記述します。
(RadikoのURL、オーディオデバイスの記述を正しく書き換えてください)

```bash:start.sh
#!/bin/bash

# Usage
# $ ./start.sh 30 'https://radiko.jp/#!/live/GBS'

# GUIセッション（Xサーバ）を指定
export DISPLAY=:0
# 音声（PulseAudio / PipeWire）の接続先を指定
export XDG_RUNTIME_DIR=/run/user/$(id -u)

echo "ARGS RAW: $1 | $2"

TIMEOUT=${1:-30}
URL=${2:-'https://radiko.jp/#!/live/GBS'}

# mpvで再生するオーディオデバイスを指定(要確認)
# $ mpv --audio-device=help
AUDIO_DEVICE="--audio-device=alsa/dmix:CARD=PJP10UR,DEV=0"

echo "TIMEOUT=$TIMEOUT"
echo "URL=$URL"

echo "Livestreaming..."
timeout --foreground --kill-after=5 "$TIMEOUT" /home/kajiru/Desktop/my_venv/bin/streamlink -p /usr/bin/mpv --player-args="$AUDIO_DEVICE" "$URL" worst
echo "Finished..."
```

完成したファイルは、次の場所に配置します。

```text:フォルダ構成
/home/kajiru/Desktop/
　├ my_venv
　└ radiko_live
　　　└ start.sh <- スクリプトを記述したファイル
```

配置した"start.sh"に実行権限を与えます。

```bash:command
# 実行権限を与える
cd /home/kajiru/Desktop/radiko_live
chmod +x .start.sh
```

実行権限を与えた後、動作確認をしておきます。
(```control + c```で停止します)

```bash:command
# ぎふちゃんを、30秒再生する
./start.sh 30 'https://radiko.jp/#!/live/GBS'
```

# CRONで再生時刻を指定する

CRONを利用して、先ほどのスクリプトを定期実行させます。
[CRON設定例](https://zenn.dev/link/comments/a442c812d2cb59)を参考にしてください。

```bash:command
# CRONを設定する
crontab -e
```

最後の行に、次の様に記述します。
(nanoで編集をしている場合は、```control + o``` で保存、```control + x``` で閉じます)

```text:cron
# 5分毎に、30秒の間ぎふちゃんを再生する(テスト用)
*/5 * * * * /home/kajiru/Desktop/radiko_live/start.sh 30 'https://radiko.jp/#!/live/GBS' >> /home/kajiru/cron_error.log 2>&1

# 毎週土曜日、18時~20時まで、7200秒の間ぎふちゃんを再生する(本番用)
0 18 * * 6 /home/kajiru/Desktop/radiko_live/start.sh 7200 'https://radiko.jp/#!/live/GBS' >> /home/kajiru/cron_error.log 2>&1
```

編集後は、次のコマンドを実行してCRONを監視しましょう。
エラーが出るときはここに表示されます。
```control + c``` で監視を中断します。

```bash:command
# CRONのログを確認する
tail -f /home/kajiru/cron_error.log
```
