# 化石化したJetsonNanoとは!?

遥か昔に購入し、数度通電しただけのJetsonNanoを発見しました。(Orinではない...)
通電したっきり化石化してしまったので、せめて"オウム返し"に挑戦します。
(性能的にギリギリらしいです!?)

![](/images/articles/3258ff3a4043a3/oumugaeshi_01.png)

今回の主人公である、JetsonNano(以下化石Jet)の性能は以下のとおりです。

|  | 内容 |
| ---- | ---- |
| CPU | Cortex-A57 ×4 |
| GPU | Maxwell 128 Core |
| メモリ | 4GB |
| Jetpack | 4.6.6 (L4T 32.7.6) |
| Ubuntu | Ubuntu 18.04.6 LTS (GNU/Linux 4.9.337-tegra aarch64) |
| User | あなたのID(ここではkajiru) |

Jetson Nano向けのJetPackは4.6.6が最終版となっており、
最新のJetPack 5系/6系へは公式にはアップグレードできません。
処理能力的にも、2026年現在では大分厳しいスペックとなります!!
(放置しすぎた結果!!)

# 1, なんとかして化石Jetに接続

先ずは、化石Jetにターミナルから操作できる様にします。
(JetsonNanoの初期設定については割愛致します)

SSH接続までの流れは以下のとおりです。
(SSH接続が出来ている方は読み飛ばしてください)

## 1-1, microUSBから直接接続

JetsonNanoには、microUSBと直接PCを繋ぐだけで、
ターミナルから直接アクセスする事ができます。

USBケーブルでJetsonNanoとMacを直接接続し、
以下のコマンドをターミナルで入力します。

```bash:USBデバイスを確認
ls /dev/cu.*
```

JetsonNanoが正しく認識されていれば、
以下の様なデバイス名が見つかるはずです。
```/dev/cu.usbmodemXXX```

見つかったデバイス名を使って、以下のコマンドで接続します。
```bash:JetsonNanoに接続
screen /dev/cu.usbmodemXXX 115200
```

## 1-2, WiFiに接続する

次に、以下のコマンドでWiFiに接続します。
(ご自宅のSSIDとPasswordで接続します)
```bash:WiFiに接続する
sudo nmcli device wifi connect "SSID" password "Password"
```

## 1-3, SSHを有効にする

SSH接続できる様にしておくと便利です。

```bash:SSH有効化
# SSH有効化
sudo systemctl start ssh
# 状態確認
sudo systemctl status ssh
```

```bash:起動時にSSHを有効にする
# 起動時SSH有効化
sudo systemctl enable ssh
# 状態確認
sudo systemctl is-enabled ssh
```

SSH接続先のhost名を確認します。

```bash:ホスト名を確認する
hostname
```

次のようなホスト名(もしくはIPアドレス)が表示されます。
```kajiru-jetsonnano```
など任意のホスト名

## 1-4, SSH接続をする

別のターミナルを立ち上げ、SSH接続します。

```bash:SSH接続する
ssh kajiru@kajiru-jetsonnano.local
```

正しいパスワードを入力してログインすると、下記の様になります。

```bash:SSH接続状態
kajiru@kajiru-jetsonnano:~$ 
```

## 1-5, FTPソフトでファイル転送

pythonで作ったプログラム等のファイルは、
[CyberDuck](https://cyberduck.io/)などのSFTPクライアントを利用すると、
GUIで簡単にファイル転送できます。

| 項目名 | 内容 |
| ---- | ---- |
| Server | kajiru-jetsonnano.local |
| Username | あなたのID(ここではkajiru) |
| Password | あなたのPassword |

# 2, マイク&スピーカーデバイスの接続

JetsonNanoのUSBにマイク&スピーカーデバイスを接続し、ターミナルから動作確認します。
(ここでのデバイスはYamaha PJP-10UR)

```bash:デバイスの確認
arecord -L
```

現在接続しているマイク&スピーカーデバイスのカード名を確認します。
```plughw:CARD=PJP10UR,DEV=0```

録音コマンドで録音テストをします。

```bash:録音
arecord -D plughw:CARD=PJP10UR,DEV=0 -f S16_LE -r 48000 -c 1 -d 5 speech.wav
```

再生コマンドで再生テストをします。

```bash:再生
aplay -D hw:2,0 speech.wav
```

録音&再生ができれば成功です。

# 3, ゆっくりボイスのインストール

軽量コンパクトな読み上げソフト、[AquesTalk](https://www.a-quest.com/download.html)から、みなさんお馴染みの「ゆっくりボイス」をお借りします。

```bash:AquesTalkインストール&再生テスト
cd Desktop
wget -qO- https://raw.githubusercontent.com/noraworld/aquestalk-installer/master/bin/pi | sh
./AquesTalkPi これは、テストです | aplay -D plughw:CARD=PJP10UR,DEV=0
```

# 4, Dockerで何もかも解決

ようやくここからが本題です。

JetsonNanoが流石に"化石化"してしまっているので、
実行環境も含めてDockerでまるっとやってしまいます。

## 4-1, Dockerの動作確認

```bash:バージョン確認
docker --version
Docker version 20.10.21, build 20.10.21-0ubuntu1~18.04.3
```

Dockerが動くかどうか確認します。

```bash:実行確認
sudo docker run hello-world
```

実行権限を"kajiru"に与えます。

```bash:権限の付与
sudo usermod -aG docker kajiru
```

## 4-2, コンテナを用意する

コンテナを作ります。
(Jetson Nano(ARM64)上でUbuntu 22.04コンテナを利用します)

```bash:コンテナを作る
docker run -it ubuntu:22.04 bash
```

Docker内で、aptのUpdateや、必要なライブラリをインストールします。
(時間がかかるので、1行づつ実行してください)

```bash:aptのUpdate、ライブラリのインストール
apt update
apt install -y python3 python3-pip ffmpeg alsa-utils
pip install faster-whisper webrtcvad requests
```

コンテナには、"コンテナID"と呼ばれるIDが付与されています。
現在のコンテナのIDを確認します。

```bash:コンテナIDを確認
docker ps
```

コンテナを保存(Commit)します。
別のターミナルを立ち上げ、先ほど確認したコンテナIDを指定してコンテナを保存します。
コンテナ名は、"jetson-whisper"とします。

```bash:コンテナの保存
docker commit <コンテナID> jetson-whisper
```

保存されたコンテナを確認します。
先ほどの、"jetson-whisper"が確認されれば成功です。

```bash:コンテナの確認
docker images
```

一度、コンテナを終了します。

```bash:コンテナを終了
exit
```

# 5, オウム返しスクリプトを用意する。

以下のスクリプトを、化石Jetの、
```/home/あなたのユーザー名/Desktop/faster_whisper/main.py```
に保存します。

:::details オウム返しスクリプト
```python:main.py
# coding: utf-8

"""
Python3
"""

#==========
# Main

import requests
import subprocess
import time
import wave
import webrtcvad
from collections import deque
from faster_whisper import WhisperModel

FILE_WAV = "/faster_whisper/sounds/speech.wav"

BUG_WORDS = [
    "ご視聴ありがとうございました"
]

def main():
    print("main!!")
    
    # WhisperModel
    # tiny < base < small...
    model = WhisperModel(
        "tiny",
        device="cpu", 
        compute_type="int8"
    )

    set_volume(50) # Volume50%
    yukkuri_speak("ウィスパーを起動するぜ!!")

    # Listen and Transcribe
    while True:
        time.sleep(1)# Sleep
        yukkuri_speak("どぞ!!")
        time.sleep(0.2)# Sleep
        listen_vad()# VAD

        # Transcribe
        yukkuri_speak("解析中...")
        segments, info = model.transcribe(
            FILE_WAV, 
            language="ja",
            beam_size=5,
            condition_on_previous_text=False
        )
        text = ""
        for segment in segments:
            print(segment.text)
            if segment.text in BUG_WORDS:
                continue
            text += segment.text + " "

        if 0 < len(text):
            yukkuri_speak(text)


def listen_vad():

    RATE_VAD = 16000 # WebRTC VADでは16000が扱いやすい
    FRAME_MS = 30
    frame_size = int(RATE_VAD * FRAME_MS / 1000) * 2

    # USB Mic(Yamaha PJP-10UR)
    proc = subprocess.Popen([
        "/usr/bin/arecord",
        "-D", "plughw:CARD=PJP10UR,DEV=0",
        "-f", "S16_LE",
        "-r", str(RATE_VAD),
        "-c", "1",
        "-t", "raw"
    ], stdout=subprocess.PIPE)

    recording     = False
    speech_count  = 0
    silence_count = 0
    audio_buffer  = []
    pre_buffer = deque(maxlen=10)

    # VAD
    # 0~3: 感度レベル
    vad = webrtcvad.Vad(1)
    while True:

        frame = proc.stdout.read(frame_size)
        if len(frame) != frame_size:
            continue

        pre_buffer.append(frame) # Ring buffer...

        if not recording:
            # Waiting...
            if vad.is_speech(frame, RATE_VAD):
                speech_count += 1
            else:
                speech_count = 0

            if 5 < speech_count:
                print("Start")
                recording = True
                silence_count = 0
                audio_buffer = list(pre_buffer)

        else:
            # Recording...
            audio_buffer.append(frame)

            if vad.is_speech(frame, RATE_VAD):
                silence_count = 0
            else:
                silence_count += 1

            if 33 < silence_count:
                print("Stop")
                proc.terminate()
                proc.wait()
                break

    # Save to File
    audio_data = b"".join(audio_buffer)
    with wave.open(FILE_WAV, "wb") as wf:
        wf.setnchannels(1)
        wf.setsampwidth(2)
        wf.setframerate(RATE_VAD)
        wf.writeframes(audio_data)


def yukkuri_speak(text):

    # AquesTalk
    aquestalk = "/aquestalkpi/AquesTalkPi" # 64bit版
    aplay = "/usr/bin/aplay"

    p1 = subprocess.Popen([aquestalk, text], stdout=subprocess.PIPE)
    #p2 = subprocess.Popen([aplay], stdin=p1.stdout) # HDMI
    #p2 = subprocess.Popen([aplay, "-D", "plughw:1,0"], stdin=p1.stdout) # ヘッドフォン
    p2 = subprocess.Popen([aplay, "-D", "plughw:CARD=PJP10UR,DEV=0"], stdin=p1.stdout) # Yamaha

    p1.stdout.close()
    p2.communicate()


def set_volume(volume):
    subprocess.run([
        "/usr/bin/amixer",
        "-c", "2",
        "sset",
        "PCM",
        f"{volume}%"
    ])


if __name__ == "__main__":
    main()
```
:::

特に、以下の部分を変える事で音声認識の精度を上げる事ができます。
tyny < base < small...
の順にモデルは軽いのですが、今回の化石Jetの場合はsmallが限界かなと感じますw

# 6, コンテナを立ち上げる

先ほど用意したコンテナ、"jetson-whisper"を立ち上げます。
オプションの意味は以下のとおりです。

| オプション | 意味 |
| ---- | ---- |
| --device | マイク&スピーカーデバイスを見つけられる様にします |
| -v | Docker内から特定のディレクトリへアクセス可能にします |

```bash:コンテナを立ち上げる(全ての行をコピーして貼り付けて実行)
docker run -it --rm \
		--device /dev/snd \
		-v /home/kajiru/Desktop/faster_whisper:/faster_whisper \
		-v /home/kajiru/Desktop/aquestalkpi:/aquestalkpi \
		jetson-whisper bash
```

コンテナを立ち上げた状態で、ゆっくりボイスの動作確認をします。
(音声が再生されれば成功です)

```bash:ゆっくりボイス
which aplay
/aquestalkpi/AquesTalkPi これは、テストです | /usr/bin/aplay -D plughw:CARD=PJP10UR,DEV=0
```

コンテナ内から、オウム返しスクリプトが見つかるかテストします。
(main.pyが見つかれば成功です)

```bash:ディレクトリ内を確認
ls /faster_whisper/
```

オウム返しスクリプトを実行します。
※初回起動時はWhisperモデル(tiny)のダウンロードが行われるため、
数分程度時間がかかる場合があります。

```bash:スクリプトを実行
python3 /faster_whisper/main.py
```

成功すると、ゆっくりボイスで「どうぞ!!」が再生されます。
直後マイクに話しかけると、「解析中...」という音声が再生されます。
しばらく待っていると解析が完了し、
結果がターミナルの文字列に、同時に音声として再生されます。
